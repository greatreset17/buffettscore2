# 本番環境安定化および Gemini 移行レポート (2026-05-02)

## 1. 発生した課題
本番環境（Vercel）へのデプロイ後、以下の重大な問題が発生しました。

*   **問題**: AI探索画面および検索機能で「Server Components render error」が発生し、アプリが即座にクラッシュする。
*   **原因**: ベクトル検索に使用していた `transformers.js` (all-MiniLM-L6-v2) ライブラリが Vercel のランタイム制限を超えており、モジュール読み込み時にクラッシュを引き起こしていた。

## 2. 実施した対策
システムの安定性を最優先し、ライブラリ依存（ローカル計算）から API 依存（クラウド計算）への完全移行を実施しました。

### A. AI モデルの刷新
*   **埋め込みモデル**: `transformers.js` を撤廃し、Google Gemini の **`gemini-embedding-001`** に移行。
*   **回答生成モデル**: **`gemini-2.0-flash`** をメインに据えつつ、API制限に強い **`gemma-4-31b-it`** をフォールバック（控え）として設定。
*   **堅牢化**: 万が一 Gemini が制限に達しても、自動的に Gemma に切り替わる「最強のフォールバックチェーン」を実装。

### B. データベースのアップグレード
*   **次元数の変更**: Gemini の高精度ベクトルに合わせ、`embedding` カラムを 384次元から **3072次元** へアップグレード。
*   **RPC関数の更新**: 3072次元でのハイブリッド検索および類似度検索に対応するよう SQL スクリプトを再定義。

### C. 運用スクリプトの整備
*   **再ベクトル化スクリプト**: Gemini API を使用して全銘柄を埋め直す `generate-embeddings-gemini.ts` を作成。
*   **安定モード**: 無料枠の API 制限（15RPM/1000RPD）を考慮し、自動待機・自動リトライ機能を搭載した安定版へ改良。

## 3. 現在のステータス
*   **本番アプリ**: **安定稼働中**。クラッシュ問題は完全に解消。
*   **AI 探索**: Gemini 制限時は Gemma が回答を担当。検索結果が 0 件でも自動で補完文章を作成するガードレールを実装済み。
*   **データ移行状況**: **1,000 銘柄** の移行が完了（2026-05-06 現在）。最新モデル **Gemini Embedding 2 (768次元)** への移行を開始し、初動の 1,000 件が完了しました。

## 4. 残り作業
*   **再ベクトル化の再開**: 本日の API 枠（RPD 1,000）をほぼ使い切ったため、**明日以降**、または別の API キーを使用して残りの約 13,000 銘柄の処理を続行してください。
*   **クォータ監視**: AI Studio で Gemini Embedding の制限がリセットされたことを確認してからスクリプトを再開することを推奨します。
